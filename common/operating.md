# 通用执行

用于所有任务的基础工程纪律。专属 skill 与本文冲突时，专属 skill 优先。

## 默认规则

- 先明确任务目标、影响范围和成功标准。
- 只改用户请求范围内的内容。
- 优先匹配现有代码、文档和子系统模式。
- 不添加推测性的抽象、选项、清理或可配置项。
- 行为变更前先定义验证方式。
- 诊断时区分事实、假设、推断和结论。
- 宣称完成前必须有本轮新鲜证据；无法验证时说明缺失证据。

## 改动前检查

1. 这是什么任务：开发、调试、审查、删除、测试、backport、性能、收尾？
2. 是否有专属 skill？如果有，先读专属 skill。
3. 需要叠加哪些通用 skill？
4. 是否会改变用户可见行为、数据、ABI/API、配置或持久化状态？
5. 用什么命令、日志、reproducer 或源码证据验证？
