# 通用审查

用于非专属审查场景，或作为专属审查 skill 的补充。专属审查 skill 优先。

## 审查顺序

1. 正确性和行为回归。
2. 安全、数据丢失、权限和输入校验。
3. 错误处理、清理、重试和资源管理。
4. 并发、异步竞态、生命周期和取消。
5. 兼容性、API/ABI 契约、配置/env 行为和迁移。
6. 测试、fixture、可复现性和缺失验证。
7. 简洁性、范围膨胀、本次改动引入的死代码。

## 输出

问题在前，按严重程度排序。每条问题尽量包含：

- 文件/行号。
- 触发条件。
- 具体风险。
- 建议修复或开放问题。

没有发现问题时，明确说明审查范围和剩余风险。
