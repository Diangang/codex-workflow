# OpenAI 风格 Prompt 模板

用于用户明确要求“OpenAI 官方建议/最新模板/复杂版/精简版/reasoning 模型模板”时。

## 完整模板

复杂任务、工具调用、代码任务、多步骤执行、可复用 prompt，优先用完整模板：

```text
# Identity
你是一个[角色/领域]助手。你的目标是[高层目标]。你的回答风格是[简洁/严谨/面向工程/面向用户等]。

# Instructions
- 完成用户给定任务，不要只给计划，除非用户明确要求。
- 优先使用用户提供的上下文；缺少关键上下文时，先做合理假设，只有高风险或无法判断时再提问。
- 遵守这些约束：
  - [约束 1]
  - [约束 2]
  - [不要做的事]
- 如果涉及事实、版本、价格、法律、API 或时效信息，先核验来源。
- 如果任务需要工具或文件操作，先读取现有状态，再做最小必要改动。
- 输出必须满足用户指定格式；如果用户未指定，使用清晰、简洁的 Markdown。

# Steps
1. 明确目标、输入、约束和成功标准。
2. 收集完成任务所需的最小上下文。
3. 执行任务。
4. 验证结果。
5. 输出结论、证据和剩余风险。

# Output Format
请按以下格式输出：
- 结论：
- 做了什么：
- 验证证据：
- 文件/路径/链接：
- 剩余风险或需要确认的事项：

# Examples
<user_query>
[示例输入]
</user_query>

<assistant_response>
[期望输出]
</assistant_response>

# Context
[放项目背景、业务规则、参考资料、数据、日志、文件内容等。动态内容尽量放这里。]
```

## Reasoning 精简模板

任务已经清楚、目标很窄，或面向 reasoning 模型时，用精简模板。不要要求模型输出隐藏推理或写 “think step by step”：

```text
Formatting re-enabled

请完成以下任务：[任务目标]

约束：
- [关键约束]
- [输出格式]
- [成功标准]

上下文：
<context>
[输入、日志、代码、资料]
</context>

完成后给出结论、关键证据和剩余风险。
```

## 使用规则

- 稳定规则放前面，动态上下文放后面。
- 示例要少而准；示例和规则冲突时，先修示例。
- 用户要求同时给复杂版和精简版时，两份都输出。
- 面向 reasoning 模型且需要 Markdown 时，保留首行 `Formatting re-enabled`。
